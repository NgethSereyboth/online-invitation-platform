# Sovan Invite Studio — usable local demo

For the complete backend demo, run `python server.py` and open `http://127.0.0.1:4175`. Opening `index.html` directly still provides local-only mode.

## Included

- Browser-based demo sign-in and multi-invitation dashboard
- Create, duplicate, and open invitation records from the dashboard
- Server-backed registration, login, 30-day sessions, and logout
- Owner-scoped drafts, publishing, uploads, versions, and RSVP management
- Archive, restore, and permanent invitation deletion controls
- Public invitation view counting and dashboard analytics
- Authenticated server material uploads with offline browser copies
- Guest-list dashboard with RSVP status summaries
- Personalized guest links that prefill the invited guest's name
- CSV guest-list export and owner-protected guest removal
- Bulk CSV guest import with quoted-field support
- Guest search, RSVP filters, and arrival check-in tracking
- Personalized QR-code preview and download
- Optional RSVP section for pure invitations that do not collect attendance responses
- Optional schedule, venue/map, gallery, and RSVP section visibility
- Editable multi-item event schedule and validated external map link
- Quick presets for pure, RSVP-focused, and full-event invitations
- Multiple named venues with individual addresses and map links
- Fine nudge and resize controls for touch and mobile editing
- Custom guest-page ordering for gallery, schedule, venue, and RSVP sections

- Editable event content with autosave through a replaceable repository layer
- Direct selection, drag, resize, duplicate, delete, font, color, and animation controls
- IndexedDB image material library (larger and safer than localStorage) and reusable canvas objects
- Responsive mobile/tablet/desktop canvas preview
- Immutable publish snapshots (draft edits do not alter the guest version until republished)
- Guest preview and persistent RSVP capture linked to the published version
- Published-version history with restore-as-draft
- JSON backup and restore for drafts, snapshots, history, and RSVP records
- Animation presets: fade up, soft zoom, slide, and continuous float

## Production integration points

The demo stores documents, snapshots, assets, and RSVPs locally. `storage.js` is the persistence boundary; for production, provide API-backed repository implementations there:

- PostgreSQL for users, draft documents, publish versions, and RSVPs
- Cloudflare R2 or Amazon S3 for original/optimized media
- Signed upload URLs generated by the backend
- Authentication provider and owner/guest authorization
- Public slug routing and CDN caching for published snapshots

## Included development backend

`server.py` uses only Python's standard library and provides SQLite-backed invitations, immutable publications, RSVP records, public routes at `/i/{slug}`, and validated material storage. Runtime data is created under `data/` and is excluded from the downloadable source archive until the server is run.

## GitHub Pages

GitHub Pages is configured to deploy directly from the root of `main`. Every push to `main` refreshes the static editor demo. GitHub Pages cannot run `server.py`, so its hosted version uses the built-in local browser storage; server publishing and cross-device RSVP persistence require a server-capable deployment later.

Uploaded data URLs are intentionally limited to 8 MB in this portable demo. Audio/video, image optimization, server validation, rate limiting, and email notifications belong in the backend implementation.
