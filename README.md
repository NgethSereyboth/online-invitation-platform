## Included development backend

`server.py` uses only Python's standard library and provides SQLite-backed invitations, immutable publications, RSVP records, public routes at `/i/{slug}`, and validated material storage. Runtime data is created under `data/` and is excluded from the downloadable source archive until the server is run.

Khmer lunar conversion uses the zero-dependency MIT-licensed [MomentKH 3.0.3](https://github.com/ThyrithSor/momentkh) browser bundle. The selected conversion result is saved into each publish snapshot so invitations retain their chosen Khmer wording.

## GitHub Pages

GitHub Pages is configured to deploy directly from the root of `main`. Every push to `main` refreshes the static editor demo. GitHub Pages cannot run `server.py`, so its hosted version uses the built-in local browser storage; server publishing and cross-device RSVP persistence require a server-capable deployment later.

Uploaded data URLs are intentionally limited to 8 MB in this portable demo. Audio/video, image optimization, server validation, rate limiting, and email notifications belong in the backend implementation.

## Collaboration and live change awareness

Shared invitations support viewer, content, designer, and manager permissions. The editor receives near-real-time remote-change notifications through authenticated Server-Sent Events (SSE) and falls back to polling when SSE is unavailable. This is intentionally conflict awareness rather than silent multi-user merge: simultaneous edits still require user review.
